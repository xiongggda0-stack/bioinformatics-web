# 公开展示型生信知识平台第一轮升级设计

## 1. 背景

Bioinformatics Web 已具备知识平台 MVP 的核心能力：

- 分析流程中心：23 个 Pipeline。
- 软件与算法库：19 个软件与算法条目。
- 数据库导航：46 个数据库资源和 8 篇教程。
- 文献集：3 篇文献。
- 跨模块关联：Pipeline、软件与算法、数据库教程和文献之间可以跳转。
- 统一搜索：支持流程、软件与算法、数据库、教程和文献检索。

下一阶段目标不是继续横向堆积内容，而是把平台打磨成可以公开展示和分享的知识平台。首轮升级优先解决安全、可信度和首次访问体验。

## 2. 目标用户

平台同时服务三类用户：

- 生信初学者：需要清晰入口、学习路径和可复制示例。
- 有经验的分析人员：需要快速检索、工具参数和流程复用。
- 实验室科研人员：需要从研究问题快速定位分析方案。

首页优先照顾初学者，但详情页保留足够的专业深度。

## 3. 本轮范围

### 3.1 包含

1. 敏感信息扫描与脱敏。
2. Pipeline 和软件与算法详情页可信度元数据。
3. 首页信息架构升级。
4. 新增学习路径页。
5. 搜索结果页体验增强。
6. 桌面端和移动端视觉检查。
7. 开发文档补充公开内容维护规范。

### 3.2 不包含

1. 管理后台。
2. 登录、角色和权限系统。
3. 在线 DAG 编辑器。
4. 智能推荐算法。
5. FASTQ 上传、任务队列和计算资源调度。
6. 生产环境 Nginx、HTTPS 和 CI/CD。

这些能力应在后续独立阶段中设计和实施。

## 4. 首页信息架构

首页不再只是模块陈列，而是帮助用户选择下一步。

### 4.1 第一屏：研究问题入口

首页第一屏使用明确问题：

> 你正在处理哪类生物信息学数据？

下方展示常用分析入口：

- Bulk RNA-seq 差异表达
- 单细胞转录组
- 可变剪接
- WGCNA 共表达网络
- CUT&Tag
- BSA
- 空间转录组
- 更多分析流程

这些入口跳转到具体 Pipeline 或带筛选条件的 Pipeline Hub。

### 4.2 第二屏：三个并列入口

1. 按研究问题找流程：进入分类化 Pipeline Hub。
2. 快速检索知识资源：进入全站搜索。
3. 从零开始学习分析：进入学习路径页面。

### 4.3 第三屏：平台分析闭环

保留并强化四步闭环：

> 选择流程 → 查看软件与参数 → 学习结果解读 → 关联文献与数据库证据

每一步均提供可点击入口。

## 5. 可信度元数据

### 5.1 公共字段

Pipeline 和软件与算法详情页统一展示：

- 验证状态：`示例验证`、`文档校验`、`待验证`。
- 最后更新日期。
- 适用范围：物种、数据类型、实验类型。
- 难度等级：入门、进阶、高级。
- 参考文献。
- 免责声明：命令和参数需要结合项目环境复核。

### 5.2 Pipeline 扩展字段

- 输入文件。
- 输出结果。
- 核心软件。
- 适用场景。

### 5.3 软件与算法扩展字段

- 软件版本。
- 安装方式。
- 官方主页或官方文档链接。
- 适用流程。
- 参数表。

### 5.4 数据存储策略

首轮优先扩展现有 JSON 元数据：

- Pipeline 使用现有 `metadata_json`。
- 软件与算法补充或复用 JSON 元数据字段。

这样可以避免首轮增加大量数据库列，并为后续后台管理保留演进空间。

前端新增可复用的 `TrustPanel` 组件，在 Pipeline 和软件详情页统一渲染可信度信息。缺失字段统一显示“待补充”，不静默隐藏。

## 6. 安全清理

公开展示前必须清理导入笔记和 seed 文档中的敏感信息。

### 6.1 扫描范围

- 密码和 Token。
- 私有下载账号。
- 邮箱。
- 内网地址。
- 本机绝对路径。

### 6.2 脱敏规则

- 用户名替换为 `<YOUR_USERNAME>`。
- 密码替换为 `<YOUR_PASSWORD>`。
- Token 替换为 `<YOUR_TOKEN>`。
- 本机路径替换为项目内相对路径或 `<YOUR_PROJECT_DIR>`。
- 无法确认用途的敏感内容先删除。

### 6.3 维护规范

在开发文档中增加：

- 禁止提交真实凭据。
- 新增或导入内容前运行敏感信息扫描。
- 示例命令必须使用占位符。

## 7. 学习路径页

新增 `/learning-paths` 页面，首轮使用前端结构化配置，不新增后端模型。

首批路线：

1. Bulk RNA-seq 入门。
2. 单细胞 RNA-seq 入门。
3. 公共数据库数据下载与复用。
4. 从流程文档到文献证据链。

每条路线由可点击步骤构成：

> 理解输入数据 → 选择流程 → 阅读关键软件参数 → 获取示例数据 → 解读结果 → 查看文献依据

步骤链接复用现有 Pipeline、软件与算法、数据库教程和文献详情页。

## 8. 搜索体验增强

在现有 `/search` 页面基础上补充：

- 搜索关键词高亮。
- 相关度排序说明。
- 热门搜索词入口。
- 结果类型筛选继续同步到 URL。

顶部搜索栏继续提供快速预览和“查看全部结果”入口。

## 9. 视觉规范

- 首页聚焦真实研究场景，不使用空泛营销文案。
- 保持紧凑、专业、适合阅读和检索的知识平台风格。
- 使用统一标签体系表达内容类型、难度和验证状态。
- Pipeline 和软件详情页统一使用可信度面板。
- 移动端导航、卡片和阅读区域不得重叠。
- 外部链接使用安全的新窗口打开方式。

## 10. 错误处理与降级

- 后端不可用时，首页统计数量显示为 `0`，基础入口仍保留。
- 学习路径链接失效时，展示明确提示。
- 可信度字段缺失时，展示“待补充”。
- 搜索接口不可用时，沿用现有错误提示。

## 11. 验证要求

1. 敏感信息扫描通过。
2. 后端回归测试通过。
3. 前端 TypeScript 检查通过。
4. 前端生产构建通过。
5. 桌面端检查：首页、学习路径、Pipeline 详情、软件详情和搜索页。
6. 移动端检查：首页、学习路径、详情页和搜索页。
7. 抽查公开外链，确认可点击并采用安全的新窗口打开方式。
